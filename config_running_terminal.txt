Para criar e instalar os APKs necessários para rodar os testes no seu dispositivo Android, execute os seguintes passos no terminal:
1. Gerar os APKs
Execute o comando abaixo para compilar tanto o aplicativo quanto o pacote de testes:
./gradlew :app:assembleDebug :app:assembleDebugAndroidTest
2. Localizar os arquivos
Os APKs foram gerados nos seguintes caminhos:
•
App Principal: app/build/outputs/apk/debug/app-debug.apk
•
App de Testes: app/build/outputs/apk/androidTest/debug/app-debug-androidTest.apk
3. Instalar no Dispositivo
Conecte seu dispositivo via USB (com o Debug USB ativado) e execute:
# Instala o aplicativo
adb install -r app/build/outputs/apk/debug/app-debug.apk

# Instala o runner de testes
adb install -r app/build/outputs/apk/androidTest/debug/app-debug-androidTest.apk
4. Executar o Teste
Agora você pode rodar o teste específico do NFC diretamente pelo adb:
Kotlin
adb shell am instrument -w -e class com.samsung.requirements.automatechecklisttest.tests.NfcTest com.samsung.requirements.automatechecklisttest.test/androidx.test.runner.AndroidJUnitRunner
Nota: O APK "genérico" que você solicitou é o app-debug.apk. Ele contém a estrutura base necessária para que o androidTest consiga se acoplar e executar as ações de UI Automator.

5. Script Facilitador (Recomendado)
Para agilizar a execução, utilize o script `run_test.sh` criado na raiz do projeto.

# 1. Dê permissão de execução (apenas na primeira vez):
chmod +x run_test.sh

# 2. Execute passando apenas o nome da classe (ou Classe#Metodo):
./run_test.sh NfcTest
./run_test.sh MobileNetworkIconTest

6. Script com Compilação (Gradle)
Para rodar testes novos ou modificados (que precisam de compilação), use o script `run_gradle_test.sh`:

# Dê permissão de execução:
chmod +x run_gradle_test.sh

# Execute passando o nome da classe:
./run_gradle_test.sh NfcTest

7. Script de Debug (ADB)
Para rodar o teste em modo de debug (o app pausa e aguarda o debugger do Android Studio):

# Dê permissão de execução:
chmod +x run_debug_test.sh

# Execute:
./run_debug_test.sh NfcTest

Peço desculpas, o comando que passei anteriormente tinha um erro comum. O pacote do instrumentation (o APK de teste) geralmente é o applicationId do seu app com o sufixo .test.

O comando adb shell am instrument correto deve usar com.samsung.requirements.automatechecklisttest.test como o nome do pacote para o AndroidJUnitRunner.

Comando Corrigido
Tente este comando. A única alteração é a adição de .test ao nome do pacote do componente de instrumentação:

adb shell am instrument -w -e class com.samsung.requirements.automatechecklisttest.tests.RegulatoryInfoTest com.samsung.requirements.automatechecklisttest.test/androidx.test.runner.AndroidJUnitRunner
Lembre-se: para este comando funcionar, os dois APKs (o do app e o de teste) precisam estar instalados no dispositivo. Se o comando acima ainda falhar, execute novamente os comandos de instalação para garantir que ambos estejam atualizados:

./gradlew installDebug
./gradlew installDebugAndroidTest
